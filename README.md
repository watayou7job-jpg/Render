# 結婚式2次会クイズアプリ

オールスター感謝祭風の早押しクイズです。参加者は各自のモバイル回線(4G/5G)からQRコードを読み取って参加します。アプリ本体はRender.com上で常時稼働させ、司会PC・投影PCはインターネットに繋がっていれば場所を問わず操作できます。

## ローカルでの動作確認・問題登録(自宅などで)

1. `start-server.bat` をダブルクリックして起動する(初回は自動で `npm install` が走ります)。
2. ブラウザで `http://localhost:3001/admin` を開き、パスワードを聞かれたら空欄のまま(ユーザー名欄)+パスワード欄に任意の文字列(未設定時は `wedding-dev-password` がデフォルト)を入力してログインする。
3. 全体設定(最低保証割合・デフォルト制限時間・配点)と、問題(問題文・任意で画像・4択・正解・制限時間・配点)を登録する。
4. `http://localhost:3001/host` を開くとパスワードを聞かれるので、同じ合言葉を入力して進行操作の動作を確認する。

## デプロイ手順(Render.com)

### 1. GitHubにコードを置く

1. https://github.com でアカウントを作成し、新しい空リポジトリを作成する(Public/Privateどちらでも可)。
2. このフォルダで以下を実行する:
   ```
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin <作成したリポジトリのURL>
   git push -u origin main
   ```

### 2. Renderにデプロイする

1. https://render.com でアカウントを作成する(GitHubアカウントでログイン可能)。
2. ダッシュボードで「New +」→「Web Service」を選択し、先ほどのGitHubリポジトリを連携する。
3. 設定項目:
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`
   - **Instance Type**: まずは Free で作成(検証後、本番1か月前を目安に Starter($7/月)へアップグレードすることを推奨。無料プランは15分無通信で自動スリープし、次回アクセス時に30〜60秒の起動遅延が発生します)。
4. 「Environment Variables」で以下を追加する:
   - `ADMIN_PASSWORD` : 司会・管理画面用の合言葉(必ず独自の値に変更する)
   - `PUBLIC_URL` は設定不要(Renderが自動で `RENDER_EXTERNAL_URL` を渡すため、アプリ側で自動的に参加者用URL・QRコードに反映されます)。
5. 「Create Web Service」でデプロイする。数分でビルドが完了し、`https://<サービス名>.onrender.com` のようなURLが発行されます。

### 3. 動作確認

- `https://<サービス名>.onrender.com/admin` にアクセスし、ADMIN_PASSWORDでログインして問題を登録する。
- `https://<サービス名>.onrender.com/host` にアクセスし、パスワードプロンプトで同じ合言葉を入力すると操作画面が使えるようになる。
- `https://<サービス名>.onrender.com/screen` を投影用PCのブラウザで開き、QRコードが表示されることを確認する。
- 実際にスマホのモバイル回線(Wi-Fiを切った状態)でQRコードを読み取り、参加〜回答〜結果発表まで一通り動作確認する。

### 重要な注意点(永続ディスクなしのため)

このデプロイ構成では永続ディスクを付けていません。そのため **サーバーが再起動すると、`questions.json`への編集内容・アップロードした画像・クラッシュ復旧用バックアップはすべて消えます**(再起動は再デプロイ時や、稀にRender側のインフラ都合で発生することがあります)。

- **問題・画像は前日までに確定させ、本番当日は再デプロイやコードの変更を行わないでください。**
- 心配な場合は、確定した `data/questions.json` の内容をこのままgitにコミットして残しておくと、万一再デプロイされても最新のコミット内容から復元されます(ただしアップロード画像は `data/uploads/` が `.gitignore` 対象のため別途手動でコミットする必要があります)。

## 進行の流れ(司会操作画面から操作)

1. 「参加受付開始」→ 参加者にQRコードを読み取ってもらう(各自のモバイル回線でOK。同じWi-Fiに繋ぐ必要はありません)
2. 参加者一覧で人数を確認し「クイズ開始」
3. 各問題: 制限時間経過で自動的に回答受付終了 →「正解発表」ボタンでタイミングを見て発表
4. 全問終了後「結果発表へ」→ 11位以下は自動スクロール→ 10位から1位までは「次の順位へ」で一人ずつ発表
5. 発表後、「結果をCSVでダウンロード」で記録を保存(こちらもADMIN_PASSWORDでの認証が必要です)

## トラブル時

- サーバーが再起動した場合、司会操作画面に「前回の途中データがあります」と表示されれば「続きから再開」を選べます(ただし上記の通り永続ディスクなしのため、再起動後は表示されないことが多い点にご注意ください)。
- 参加者のスマホがリロード・再接続しても、名前登録済みならスコアと状態は自動的に復元されます。
- 司会操作画面でボタンを押しても反応しない場合、パスワード認証が通っていない可能性があります。ページを再読み込みしてパスワードを入力し直してください。
